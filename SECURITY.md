# Security

- **Do not post secrets, API tokens, or passwords** in issues or pull requests. Use environment variables and reference `.env.example` in documentation.
- If you discover a vulnerability that could affect users (e.g. credential handling, script injection), please report it privately: open an issue with a minimal description and mark it as sensitive, or contact the maintainers via the email or contact method listed in the repo profile. Do not disclose sensitive details in public issues.
